================================================================================
                    CRITICAL CACHING ISSUES - QUICK START
================================================================================

YOUR PROBLEM:
├─ Users see stale products/categories
├─ Clearing cookies/cache temporarily fixes it
├─ Issue repeats after 5 minutes
└─ Happens on all devices (web/mobile/tablet)

ROOT CAUSE (In Order of Impact):

 1. SESSION STORAGE CACHE VERSIONING BUG (ProductContext.tsx)
    Old cache keys never properly invalidate
    Result: Stale products served repeatedly

 2. THREE OVERLAPPING CACHE SYSTEMS (caching.ts, newCaching.ts,
    ProductContext.tsx, hybridCache.ts)
    No single source of truth
    Result: Poison data in one system affects all

 3. SERVICE WORKER HTTP CACHING (main.tsx, App.tsx)
    Browser HTTP cache (1 hour) not controlled
    Result: Data poisoned even after session cache clears

 4. SILENT CACHE FAILURES (all caching files)
    try { ... } catch { /* ignore */ }
    Result: No error detection, just stale data

 5. BROWSER HTTP CACHE NOT CONTROLLED
    Supabase responses cached for 1+ hour
    Result: Network requests bypassed entirely

================================================================================
                            QUICK WINS (30 MINUTES)
================================================================================

PHASE 1: Clear Poisoned Cache on App Load
File: src/main.tsx
Add before React renders (before createRoot):

    if (typeof window !== 'undefined') {
      const keysToRemove = [];
      for (let i = 0; i < sessionStorage.length; i++) {
        const key = sessionStorage.key(i);
        if (key?.startsWith('pc_') || key?.startsWith('perfume_cache_')) {
          keysToRemove.push(key);
        }
      }
      keysToRemove.forEach(key => sessionStorage.removeItem(key));
    }

Time: 5 minutes
Impact: 60% improvement
